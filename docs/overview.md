# SignVani — Project Overview

## What is SignVani?

SignVani is a web-based toolkit for **Indian Sign Language (ISL)** that bridges the communication gap between the deaf community and hearing individuals. The name combines "Sign" with "Vani" (Sanskrit for *voice/speech*), reflecting its mission to give a voice to sign language.

The system converts spoken and written English into animated ISL gestures, displayed through a **3D avatar** in the browser. It also provides tools for learning ISL and creating shareable ISL videos.

---

## Problem Statement

India has an estimated 18 million deaf and hard-of-hearing individuals who rely on Indian Sign Language (ISL) as their primary mode of communication. Most hearing people have no knowledge of ISL, creating a significant communication barrier. Existing tools for ISL are either limited in vocabulary, require specialist hardware, or are not accessible as web applications.

SignVani addresses this by providing a free, browser-based toolkit that works on consumer hardware — including low-power devices like the Raspberry Pi 4.

---

## Core Features

| Feature | Description |
|---|---|
| **Text to ISL** | Convert typed English text into animated ISL gestures on a 3D avatar |
| **Speech to ISL** | Record speech via microphone; the system transcribes it and plays the ISL animation |
| **Learn ISL** | Interactive interface to learn the ISL alphabet (A–Z) and 48 common words |
| **Create Video** | Compose an ISL video from text or speech and save it with a unique ID |
| **Video Gallery** | Browse, search, and watch all saved ISL videos |
| **NLP Backend** | Python FastAPI backend that performs linguistic processing — grammar transformation, gloss mapping, and HamNoSys generation |

## ISL Linguistic Pipeline

SignVani implements the following linguistic transformations to convert English to ISL:

1. **Tokenization** — Split input into words using NLTK
2. **POS Tagging** — Tag each token (noun, verb, adjective, etc.)
3. **Lemmatization** — Reduce words to base forms (e.g., "running" → "run")
4. **Stopword Removal** — Remove filler words not used in ISL
5. **SVO → SOV Transformation** — English uses Subject-Verb-Object order; ISL uses Subject-Object-Verb order
6. **Gloss Mapping** — Map English lemmas to ISL glosses via SQLite database
7. **HamNoSys Lookup** — Retrieve HamNoSys phonological codes for each gloss
8. **Animation Generation** — Convert HamNoSys codes into Three.js bone transformation keyframes

## 3D Avatar Animation System

The frontend uses **Three.js** to render a rigged 3D humanoid avatar (xbot or ybot models). Animations are driven by **bone transformations** — each sign is encoded as a sequence of rotation/position changes on named skeleton bones.

Two animation modes exist:
- **Built-in animations** — Pre-defined word and alphabet animations stored in `wordsData.json` and `Animations/Alphabets/`
- **Backend-driven animations** — Keyframe data generated by the NLP backend from HamNoSys codes, consumed via `POST /api/text-to-handsign`

## Target Hardware

The system is specifically optimized for deployment on a **Raspberry Pi 4B**:
- Three.js renderer uses medium precision shaders, disabled antialiasing, and 1:1 pixel ratio
- Shadow rendering is disabled on all 3D models
- The NLP backend uses a small Vosk model, connection-pooled SQLite, and LRU-cached gloss lookups
- Audio buffering is tuned to ~64ms per chunk at 16kHz
